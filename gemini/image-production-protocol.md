# 🎨 통합 이미지 생성 프로토콜 (Global Image Production Protocol)

이 문서는 모든 퀴즈 및 게임북 컨텐츠의 시각적 일관성을 유지하고, 초상권 및 저작권 문제를 방지하기 위한 AI 에이전트의 핵심 지침입니다.

## 1. 실존 인물 생성 금지 (Strict No-Real-Person Rule)

**가장 중요한 규칙입니다.** 모든 생성 이미지는 특정 실존 인물이나 배우를 연상시켜서는 안 됩니다.

*   **실명 사용 금지**: 프롬프트에 연예인, 운동선수, 정치인 등 실존 인물의 이름을 절대 포함하지 마십시오.
*   **특징 모사 금지**: 특정 배우의 목소리나 외모 특징을 기반으로 한 묘사를 피하십시오.
*   **제네릭 캐릭터 사용**: "A generic middle-aged man", "An anonymous detective"와 같이 일반적이고 추상적인 묘사를 사용하십시오.
*   **초상권 보호**: 생성된 이미지가 특정 인물과 지나치게 닮았다면 즉시 폐기하고 다시 생성하십시오.

## 2. 스타일 고정 및 일반화 (Stylization)

실사(Photorealistic)보다는 예술적인 스타일을 우선적으로 사용하여 초상권 문제를 원천 차단합니다.

*   **권장 스타일**: 3D 애니메이션, 수채화, 픽셀 아트, 일러스트레이션 등.
*   **실사 스타일 지양**: 실사 이미지가 필요하더라도 "Cinematic drama style" 정도로 표현하되, 인물보다는 주변 환경과 조명에 집중하십시오.
*   **배경 위주 구성**: 인물의 얼굴이 정면으로 크게 부각되는 구도보다는, 뒷모습이나 측면, 배경과 조화를 이루는 구도를 권장합니다.

## 3. 프롬프트 구조 가이드라인

프롬프트 작성 시 다음 순서를 준수하십시오:
1.  **화풍(Style)**: (예: 3D Pixar style, Studio Ghibli inspired watercolor)
2.  **주요 장면(Subject)**: (예: A detective investigating a dimly lit room)
3.  **조명 및 분위기(Mood)**: (예: Moody lighting, mysterious atmosphere)
4.  **부정 프롬프트(Negative Prompt - 툴이 지원하는 경우)**: Text, letters, numbers, celebrity, real person.

## 4. 품질 및 일관성 체크리스트

- [ ] 이미지에 글자나 숫자가 포함되어 있지 않은가?
- [ ] 특정 실존 인물이나 배우가 떠오르지 않는 제네릭한 모습인가?
- [ ] 같은 컨텐츠 내의 다른 이미지들과 화풍이 100% 일치하는가?
- [ ] 해상도는 512x512(또는 지정된 규격)를 준수했는가?

---
**주의**: 위 규칙을 위반하여 생성된 컨텐츠는 배포할 수 없습니다. 유저가 '실제 배우 같다'고 느끼는 이미지는 즉시 수정하십시오.
